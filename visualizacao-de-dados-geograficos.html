Visualização de dados geográficos heterogêneos para suporte à tomada de decisões na gestão de risco de inundações

Tema: Visual Analytics

Lacuna/problema: Visualização de dados de contexto geográfico de fontes distintas

Hipóteses e Objetivo:
Hipóteses:
  1. Dados de contexto geográfico não são combinados com mídias sociais em Sistemas de Informações Geográficas
  2. As interfaces de visualização de dados das agências de emergências não são suficientes para prevenção e previsão de eventos naturais

Objetivo Geral: Desenvolver uma abordagem de visualização de dados geográficos integrados, vindos de fontes heterogêneas, para o suporte à tomada de decisões na gestão de risco de inundações

Objetivos Específicos:
  1. Desenvolver um método de integração de fontes heterogêneas (dados oficiais, redes de sensores, informações geográficas voluntárias e mídias sociais) em uma base unificada de informações
  2. Desenvolver uma interface interativa para representação visual de dados geográficos heterogêneos
  3. Avaliar experimentalmente a abordagem de visualização
  4. As representações visuais de dados geográficos heterogêneos auxiliam no processo de tomada de decisão na gestão de risco de inundações

Justificativa/motivação: Conforme os dados disponibilizados pelo Centre for Research on the Epidemiology of Disasters (EM-DAT, 2009), as inundações no Brasil foram os eventos naturais que mais afetaram pessoas e causaram mortes entre o período de 2004 e 2014

Resumo com Estruturação Explícita das partes componentes:
Contexto: Inundações são frequentes e impactantes no mundo, causando mais prejuízos econômicos do que outros eventos naturais.
Lacuna: A integração de dados de fontes heterogêneas tem um grande potencial para a descoberta de conhecimento para mitigar riscos.
Propósito: Para minimizar os impactos causados e aumentar a resiliência de comunidades vulneráveis contra esse desastre, Sistemas de Informação Geográfica monitoram dados geoespaciais de redes de sensores, mídias sociais, dados oficiais e de informações de voluntários, contribuindo com a gestão de risco de inundações.
Metodologia: Este projeto de pesquisa propõe uma abordagem de visualização interativa de dados geográficos heterogêneos para o suporte à tomada de decisões na gestão de risco de inundações.

Metodologia de Desenvolvimento dos métodos (ou dos sistemas):
O escopo deste projeto é fundamentado na estrutura do escopo de analítica visual, composto por coleta de dados de fontes heterogêneas, pré-processamento dos dados, interação com informações e descoberta de conhecimento útil. Contudo, o componente de coleta de dados não está incluído no escopo, pois a plataforma AGORA disponibiliza dados geográficos de relatórios oficiais, redes de sensores, informações geográficas voluntárias e mídias sociais.
O desenvolvimento do projeto deve ser incremental e iterativo, permitindo que pesquisadores e profissionais de agências de emergência (por exemplo, CEMADEN) acessem, explorem e comentem melhorias na plataforma, enquanto o processo de desenvolvimento está em andamento.

Desenvolvimento de método de transformação de dados
Os dados coletados pelos componentes da plataforma AGORA estão em bancos de dados e em serviços geoespaciais de sensores padronizados pela Open Geospatial Consortium - Sensor Observation Service -, possibilitando a consulta de dados captados pelas redes de sensores.
Por meio de conexões com os bancos ou requisições aos serviços, os dados consultados passarão por métodos de fusão de dados, facilitando o uso dos dados pelos outros componentes deste projeto. Os métodos de fusão de dados serão escolhidos através de revisões sistemáticas de literatura.

Desenvolvimento de interface de visualização
Para facilitar o uso concorrente e colaborativo da interface, esta deve ser uma aplicação web, com suporte aos navegadores mais usuais. Sua arquitetura computacional cliente-servidor permitirá que o servidor seja responsável pelo processamento de informação, enquanto o cliente exibe e manipula gráficos e mapas interativos.
A interface terá como funcionalidade principal a representação interativa de dados, de maneira individual e correlacionada. Individualmente, cada fonte de dados terá uma visualização específica com filtros de acordo com seu contexto de origem (por exemplo, filtro de mídias sociais por palavra-chave). Ao correlacionar as múltiplas fontes, os dados geográficos serão representados em mapas interativos com informações relevantes da fonte escolhida, gerando uma representação geográfica com cruzamento de dados (por exemplo, mapa de áreas de risco de inundação baseado em dados de sensores móveis).

Metodologia de Avaliação:
Devido à metodologia de desenvolvimento ágil aplicada no desenvolvimento da interface de visualização de dados geográficos, haverá intervalos onde pesquisadores e membros de agências de emergência utilizarão esta aplicação. As avaliações desses profissionais auxiliarão na definição da ordem de implementação dos requisitos funcionais, entregando primeiro os fundamentais.
Além dessa forma de avaliação da interface, ferramentas de analítica web, como a plataforma open source Piwik, serão responsáveis por monitorar a interface de visualização de dados, registrando as ações realizadas pelos usuários, como: origem do acesso, engajamento (tempo no site e páginas visitas) e rastreamento de cliques.
Com a análise dessas informações, será possível aprimorar a experiência do usuário na plataforma e ajustar os métodos de transformação e visualização de dados.
